Compiling the Notes

Due to the frequency of changes to the notes, I no longer provide them in PDF format. Whilst I have written code to compile the .tex files automatically, sadly it cannot be run on the DoC web servers (since they don't have pdflatex). You can however compile them yourself on any lab machine.

Errors, Corrections, Suggestions and Extensions

If you use these notes, please help me to improve them.

I have recently made the LaTeX source for all my notes openly available on GitHub. If you submit an issue or a pull request, you should find yourself adequately rewarded in nerd points:

  • The first finder of any error (submit an issue) will receive a tidy sum of 0.32 nerd points.
  • If you then provide a correction (submit a pull request), you will receive an additional 0.32 nerd points.
  • A valuable suggestion (submit an issue) is worth 0.08 nerd points.
  • A minor extension (submit a pull request) is worth 0.16 nerd points.
  • If you provide a significant extension (submit a pull request) to the notes, I will give you a full 1.28 nerd points.

I will similarly reward the first individuals who find and correct errors (drop me a quick email) in past paper solutions with 0.64 nerd points.
